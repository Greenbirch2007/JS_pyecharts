因为uwsgi部署失败，这次用nginx +Gunicorn 进行的部署

（1）安装
首先安装Gunicorn 

pip install gunicorn

再安装nginx ，见收集的文档


（2）调试，修改配置

其中修改nginx

 /usr/local/nginx/conf

vi nginx.conf 



 #gzip  on;
# 增加
proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;  

server {  
listen       80;  
 server_name  139.162.19.43;
 location / {  
 #root   html;  
 #index  index.html index.htm;  
 proxy_pass http://127.0.0.1:8000; 
access_log /home/admin/api_access.log;
 proxy_read_timeout 60;  
 error_log  /home/admin/api_error.log;}

注意点：
1. /home/admin/api_error.log这个文件因为没有所以会报错
要么删掉这个配置项，要么就创建文件

2. 最终访问好像都没有用到8000端口！


配置好nginx后，直接使用


gunicorn -w 4 -b 127.0.0.1:8000 _3m_flaskW:app


等于是浏览器-----nginx----8000-----gunicorn----app
